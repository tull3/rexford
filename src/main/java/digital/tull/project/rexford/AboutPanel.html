Some Details

  I'm assuming that you're like me, and when you click a link that says "About," you want some technical details. Very well. (Warning: bad security practice ahead.) Let's start with the Java stuff. This subdomain is run by a really simple web app that I like to call "Rexford." Rexford, for those of you who are interested, is named after one of NASCAR's first cup champions. It is built with the Wicket framework. Unlike the other Java web framework that I learned at first, JavaServer Faces, Wicket is more of POJO API, uses very little server-level configuration, and none of the Java EE spec. Without going into the whole backstory, this essentially has helped me to learn the framework much more thoroughly. Also, there is no built-in JPA support, so this forced me to get really cozy with JDBC and set up the database connections manually. Rexford primarily uses an embedded Derby database to store the articles and metadata, but PostgreSQL is being integrated more and more into my work. It will serve the bulk of the data as Rexford grows in complexity. Right now, Rexford, Tomcat, and Derby live on a Fedora server with a SSD, and PostgreSQL lives on a RAID 1 openSUSE Leap server with development versions of Rexford. There is also an Arch system on the network which is used as a FTP, Samba, and Rsync server.
